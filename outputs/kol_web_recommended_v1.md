# KOL 合作资料网页 V1 推荐定稿版

这份文档不是“备选方案集合”，而是一份已经帮你做过取舍的 `推荐版 V1`。

如果要尽快开始做网页，我建议第一版就按这份来。

---

## 1. 我建议直接采用的方案

### 1.1 首页结构

采用 4 个主入口：

- `Dashboard`
- `达人库`
- `竞品品牌库`
- `汇报导出`

不单独把“新增达人”做成菜单，而是作为达人库页右上角的主按钮。

### 1.2 达人库默认视图

建议默认使用 `表格视图优先`。

原因：

- 你后续一定会管理越来越多达人
- 你最常做的是筛选、改状态、勾选导出
- 这些操作表格视图效率明显更高

同时保留 `卡片视图切换`，用来看截图和审美风格。

### 1.3 达人详情页结构

建议采用：

- 左侧：资料与合作信息
- 右侧：截图资料墙

原因：

- 这是最符合你实际判断达人的方式
- 一眼就能同时看到“资料”和“审美图”
- 比上下长页面更适合频繁查看

### 1.4 状态系统

建议第一版统一只保留下面这套状态：

- `未联系`
- `已联系未回复`
- `已回复沟通中`
- `待报价`
- `已报价待确认`
- `已寄样`
- `合作进行中`
- `已发布内容`
- `已完成`
- `不合作`

这套状态已经足够覆盖你现在 Excel 里的工作流，而且后续做筛选、统计、导出都很顺。

### 1.5 新增达人方式

建议第一版做成 `手动粘贴链接 + 系统自动填充资料`。

新增时由你填写：

- 平台
- 达人链接

系统自动尝试补全：

- 达人账号
- 达人名称
- 头像 / 封面
- 平台识别
- 简介 / 关键词
- 粉丝数
- 内容类型

如果自动抓取不完整，再允许你手动补充和修改。

原因：

- 这是你明确需要的核心体验
- 能显著减少重复录入
- 比“全手动”更接近真实使用场景

### 1.6 删除达人方式

建议第一版不做真正物理删除，而是做 `软删除留痕`。

删除后的显示规则：

- 在达人库中保留该行
- 整行显示为 `红色 + 删除线`
- 状态标记为 `已删除`
- 记录删除时间和删除原因

原因：

- 方便回溯历史
- 避免误删后找不回来
- 很符合你管理达人池时“删了也要留痕”的需求

### 1.7 汇报导出优先级

建议第一版优先做：

1. `简介表格版汇报页`
2. `浏览器打印/PDF`

原因：

- 这是你已经明确拍板的输出形式
- 领导查看时更快扫完一批达人
- 后续如果需要，再补图文卡片版

### 1.8 竞品品牌页

建议第一版字段在你要求基础上，额外加 2 个字段：

- `品类`
- `价格带`

完整字段建议为：

- 品牌名字
- 品牌介绍
- 品牌头像/logo
- Amazon 网站
- 官网
- 母公司
- 归属国家
- 品类
- 价格带
- 发现来源
- 关联达人
- 备注

## 2. 我建议的 V1 页面定稿

## 2.1 Dashboard

只做轻量看板，不做复杂分析。

展示：

- 总达人数量
- 各平台数量
- 未联系数量
- 已联系未回复数量
- 合作中数量
- 已完成数量
- 待汇报数量
- 最近新增达人
- 最近更新达人

这页的目标只有一个：让你和领导一打开就知道盘子情况。

## 2.2 达人库

这是 V1 的主页面，建议把 80% 精力放在这里。

默认表格列：

- 勾选框
- 平台
- 达人账号
- 达人名称
- 国家
- 粉丝数
- 粉丝级别
- 是否联系
- 合作状态
- 优先级
- 推荐产品
- 最近更新时间
- 操作

顶部筛选：

- 搜索
- 平台
- 国家
- 是否联系
- 合作状态
- 优先级
- 粉丝级别
- 是否加入汇报名单

顶部按钮：

- `+ 新增达人`
- `导出当前筛选结果`
- `加入汇报名单`
- `查看已删除`

## 2.3 达人详情页

V1 详情页应该具备：

- 基础资料编辑
- 合作信息编辑
- 联系记录时间线
- 截图资料墙
- 数据表现记录

其中最重要的是：

- `上传截图`
- `设置封面图`
- `记录状态变化`
- `自动回填达人资料`

因为这三件事最能替代你现在“Excel + 文件夹截图”的分散工作流。

## 2.4 竞品品牌库

V1 用卡片视图为主，表格视图为辅。

原因：

- 品牌信息更适合卡片浏览
- logo、国家、官网入口都能直接看到
- 后续领导看也更直观

## 2.5 汇报导出页

V1 先只做一种模板：

- `简介表格版`

原因：

- 这是你已经确认的版本
- 最适合快速汇报和打印导出
- 第一版实现成本更稳

每个达人表格列展示：

- 达人名
- 平台
- 国家
- 粉丝量
- 当前状态
- 推荐理由
- 原始主页链接

## 3. 第一版必须完成的功能

我建议把 V1 范围卡死在下面这些，不然很容易越做越重：

### 必须做

- 达人列表页
- 达人详情页
- 新增达人并自动填资料
- 编辑达人
- 上传截图
- 封面图设置
- 状态更新
- 删除留痕
- 多条件筛选
- 勾选加入汇报名单
- 竞品品牌库页面
- 导出汇报页为 PDF

### 先不做

- 自动抓取 Instagram / TikTok 信息
- 自动识别头像
- 自动抓取粉丝数
- 自动发邮件 / DM
- 权限系统
- 多人协作审批流
- 高级 BI 报表

这能保证第一版足够轻、足够快，而且已经实用。

## 4. 你现有 Excel 的导入建议

我建议第一版采用：

- `先人工导入核心数据`
- `后续再做 Excel 批量导入`

### 为什么仍然不建议第一版先做复杂 Excel 导入

- 你现在两份 Excel 结构相近但不完全一致
- 还有截图、话术、补充备注这些非结构化内容
- 第一版的自动化优先级应该放在“新增单个达人时自动补全”，而不是“先做超重的历史导入器”

### 我建议的导入顺序

第一批先导入达人主信息：

- 平台
- 账号
- 名称
- 链接
- 国家
- 粉丝数
- 粉丝级别
- 内容类型
- 匹配理由
- 推荐产品
- 联系状态
- 报价
- 备注

截图和竞品品牌可以在系统跑起来后慢慢补。

## 5. 预计开发节奏

如果按这个 V1 做，我建议分 4 步：

### 第 1 步

先搭网页框架：

- 导航
- 页面布局
- 假数据跑通

### 第 2 步

做达人库：

- 列表
- 筛选
- 新增
- 编辑

### 第 3 步

做达人详情：

- 截图上传
- 状态流转
- 联系记录

### 第 4 步

做竞品品牌库和汇报导出

这样推进最稳，也最贴近你真实使用顺序。

---

## 6. 最终我建议你现在就拍板的版本

如果你现在让我直接开始开发，我会按下面这版执行：

- 先做本地网页，不做线上账号系统
- 默认表格视图管理达人
- 每个达人有单独详情页
- 详情页右侧重点展示截图资料墙
- 支持四个平台分类
- 支持按国家、是否联系、合作状态筛选
- 支持粘贴达人链接后自动填资料
- 支持修改是否联系和合作情况
- 支持删除留痕，删除项红色加删除线保留
- 支持勾选达人生成领导汇报名单
- 支持单独的竞品品牌库页面
- 竞品品牌页包含品类、价格带
- 汇报导出优先做简介表格版 PDF

这版最贴近你现在的真实需求，也最适合作为第一版落地。
