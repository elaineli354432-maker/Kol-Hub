# KOL 达人合作资料管理网页方案

## 1. 项目目标

搭建一个内部使用的网页工具，用来统一管理 Brandream 在 `TikTok`、`Instagram`、`Facebook`、`Pinterest` 四个平台上的达人合作资料，同时支持：

- 持续新增达人链接
- 粘贴链接后由系统自动填写可识别资料
- 记录达人是否联系、合作进度、报价、寄样、内容产出情况
- 存放达人主页截图、参考视频截图、审美参考图，并标注对应 KOL
- 按国家、是否联系、合作状态等条件快速筛选
- 勾选一批达人生成简介表格版汇报，导出给领导
- 单独维护一个竞品品牌资料页，沉淀品牌研究信息
- 删除达人时保留删除痕迹，而不是直接消失

这个工具本质上是一个结合了 `达人库`、`素材墙`、`联系跟进表`、`领导汇报清单`、`竞品品牌库` 的轻量业务系统。

## 2. 已确认的 V1 方向

你已经确认的关键决策如下：

1. 可以随时增加达人，并且由系统自动填写资料
2. 达人库主页默认使用表格
3. 合作状态使用文档中的 10 个统一状态
4. 领导导出使用简介表格版
5. 竞品品牌页增加 `品类`、`价格带`
6. 删除达人需要留痕，表现为 `红色 + 删除线`

## 3. 适用人群与使用场景

主要使用人：

- 你自己：持续找达人、补资料、截图、改合作状态
- 内部同事：查看达人资料、筛选可合作名单
- 领导：查看当前重点达人名单和竞品品牌信息

核心使用场景：

1. 发现一个达人，粘贴主页链接，新建到系统
2. 系统自动回填账号、名称、头像、粉丝数等信息
3. 你补充国家、判断理由、推荐产品、备注
4. 给这个达人上传主页截图和风格参考截图
5. 根据国家、平台、合作状态筛出一批达人
6. 勾选一批达人生成简介表格版汇报清单
7. 在找达人过程中，把看到的竞品品牌记录进品牌库
8. 删除不再需要的达人，但保留删除痕迹和原因

## 4. 信息架构

建议网页做成 4 个主入口：

1. `Dashboard`
2. `达人库`
3. `竞品品牌库`
4. `汇报导出`

其中 `达人详情` 不是顶层菜单，而是从达人库点击进入。

## 5. 页面方案

### 5.1 Dashboard

用途：一打开就能看整体进度。

建议展示模块：

- 总达人数
- 各平台达人数量
- 未联系人数
- 已联系未回复人数
- 已回复人数
- 合作中人数
- 已完成合作人数
- 待汇报达人数量
- 本周新增达人
- 最近更新达人

### 5.2 达人库

用途：主工作台，日常大部分操作都在这里完成。

默认使用 `表格视图`，并保留卡片视图切换。

#### 列表页顶部区域

- 搜索框：支持搜索达人名、账号、链接、关键词
- 平台筛选：`TikTok / Instagram / Facebook / Pinterest`
- 国家筛选
- 是否已联系筛选：`全部 / 已联系 / 未联系`
- 合作状态筛选
- 优先级筛选
- 粉丝级别筛选
- 是否已加入汇报名单筛选
- 添加达人按钮
- 导出按钮
- 查看已删除按钮

#### 表格列建议

- 勾选框
- 平台
- 达人账号
- 达人名称
- 国家/地区
- 粉丝数
- 粉丝级别
- 内容类型
- 是否已联系
- 合作状态
- 合作方式建议
- 建议优先级
- 报价
- 最近更新时间
- 操作

#### 列表页支持的快捷操作

- 直接改 `是否已联系`
- 直接改 `合作状态`
- 直接改 `优先级`
- 打开达人详情
- 打开原始主页链接
- 加入汇报名单
- 删除留痕达人

#### 删除留痕规则

- 删除达人后，不从列表物理移除
- 默认列表不展示已删除达人
- 切换到“查看已删除”时，已删除达人显示为 `红色 + 删除线`
- 系统记录删除时间
- 系统要求填写删除原因

### 5.3 达人详情页

用途：沉淀单个达人的完整资料，是最核心页面。

建议分成 5 个区块：

#### A. 基础信息

- 平台
- 达人账号
- 达人名称
- 主页链接
- 国家/地区
- 粉丝数
- 粉丝级别
- 内容类型
- 主页简介/关键词

#### B. 合作判断

- 是否适合 Brandream
- 匹配理由
- 推荐产品
- 合作方式建议
- 建议优先级

#### C. 联系与合作进度

- 初次联系渠道
- 联系方式
- 是否已联系
- 合作状态
- 计划联系日期
- 实际联系日期
- 报价
- 寄样地址
- 发布时间
- 内容链接
- 折扣码
- 点击
- 下单
- 销售额
- 备注/待确认

#### D. 截图资料墙

截图分 3 类：

- `达人主页截图`
- `达人作品截图`
- `参考审美截图`

每张图建议带这些信息：

- 图片预览
- 图片标题
- 绑定达人名
- 图片分类
- 图片说明
- 上传时间
- 上传人

额外建议：

- 支持拖拽上传
- 支持把某张图设为“封面图”

#### E. 联系记录时间线

建议记录：

- 什么时候添加达人
- 什么时候自动回填资料
- 什么时候发了 DM / 邮件
- 什么时候收到回复
- 什么时候拿到报价
- 什么时候寄样
- 什么时候发布内容
- 什么时候被删除及删除原因

### 5.4 新增达人页/弹窗

用途：快速录入一个新达人。

建议最少必填字段：

- 平台
- 达人链接

系统自动回填字段：

- 达人账号
- 达人名称
- 平台识别结果
- 头像 / 封面图
- 粉丝数
- 简介 / 关键词
- 内容类型

建议人工补充字段：

- 国家/地区
- 联系方式
- 匹配理由
- 推荐产品
- 备注

第一版就建议支持“粘贴链接后自动填资料，再人工修正”的模式。

### 5.5 竞品品牌库

用途：记录在找达人过程中发现的竞品品牌。

建议页面形式：

- 列表页
- 品牌详情弹窗或详情页

#### 品牌字段建议

- 品牌名称
- 品牌介绍
- 品牌头像 / logo
- Amazon 网站
- 官网
- 母公司
- 归属国家
- 品类
- 价格带
- 发现来源
- 关联达人
- 备注

#### 建议支持的筛选

- 国家
- 品类
- 价格带
- 是否有 Amazon
- 是否有官网
- 母公司

### 5.6 汇报导出页

用途：把你筛选或勾选出来的一批达人整理成可发领导的资料页。

第一版默认输出 `简介表格版`。

#### 生成方式

1. 从达人库中手动勾选加入
2. 根据筛选结果一键生成

#### 汇报表格列建议

- 达人名
- 平台
- 国家
- 粉丝量
- 合作状态
- 推荐理由
- 链接

#### 导出形式建议

- `打印版网页`
- `导出 PDF`
- `导出 Excel`

### 7.4 平台分类

- Instagram

## 10. 技术实现建议

如果后面开始开发，我建议用下面这套：

- 前端：`Next.js`
- UI：`Tailwind CSS + shadcn/ui`
- 数据库：`SQLite`
- 文件上传：本地文件夹存储
- 导出：`PDF + Excel`

## 11. 结论

这套网页第一版的核心目标，不是做一个复杂 CRM，而是先把你现在分散在 Excel、截图、链接、备注里的工作流收拢到一个地方，重点提升 5 件事：

- 找达人后能快速沉淀资料
- 粘贴链接后系统自动填资料
- 截图和达人能绑定在一起
- 跟进状态能持续更新
- 给领导汇报时能快速输出简介表格版

这个方案已经可以直接进入开发阶段。
